# Evotown 像素风格 UI 设计规范

> 严格参考 FC/NES 吞食天地2 大地图风格，逐像素复刻
>
> 核心元素详细定义见 → [PIXEL_CORE_ELEMENTS.md](./PIXEL_CORE_ELEMENTS.md)

---

## 一、总体风格约束

### 必须遵守
- 所有渲染使用 **NES 受限调色板**（见核心元素文件）
- 大面积区域使用**有序抖动**（ordered dithering）代替渐变
- 所有精灵/对象使用 **1px #000000 纯黑轮廓**
- 整数倍缩放（2x/3x/4x），`image-rendering: pixelated`
- 单个精灵颜色数 ≤ 6（含轮廓黑）

### 绝对禁止
- 渐变、模糊、抗锯齿、阴影、半透明
- 分数缩放、亚像素渲染
- 超出调色板的颜色

## 四、角色精灵 (Character Sprites)

> 详细像素结构见 [PIXEL_CORE_ELEMENTS.md](./PIXEL_CORE_ELEMENTS.md)

- **尺寸**: 16×16px
- **配色限制**: 每角色 4-5 色 + 黑色轮廓
- **结构**: 头部(头盔+面部) 5-6px + 身体(铠甲) 10-11px
- **特征**: 蓝色头盔标识身份, 白色铠甲+黑横纹, 武器 1px 线延伸
- **动画**: 站立1帧, 行走2帧循环

## 七、场景架构

```
BootScene     → 初始化、像素字体预加载
PreloadScene  → 程序化纹理生成、进度条
TownScene     → 主场景（地形/建筑/角色）
```

### 图层顺序（自底向上, z-index递增）
1. **地形层** — 草地/道路 Tile 铺设（程序化抖动纹理）
2. **装饰层** — 山脚岩石、散落小树
3. **森林/山脉层** — 密集森林、山脉群（前后层叠遮挡）
4. **建筑层** — 城镇建筑
5. **角色层** — 玩家/NPC（按 y 轴排序）
6. **UI 覆盖层** — 标题、对话框、菜单

## 九、字体规范

- **像素字体**: Press Start 2P / VT323 / Silkscreen
- **字号**: 8px (游戏内对话) / 10px (菜单) / 6px (标注)
- **渲染**: 禁用字体平滑, `font-smooth: never`
- **React UI**: 同上像素字体, `text-rendering: optimizeSpeed`

## 十、React UI 像素化规则

| 规则 | 做法 |
|------|------|
| 圆角 | ❌ 禁用 `border-radius`, 全部直角 |
| 阴影 | ❌ 禁用 `box-shadow`, 用 1-2px 偏移纯色块模拟 |
| 边框 | 1px 实线, #F8F8F8 或 #000000 |
| 按钮 | 矩形硬边, 纯色填充, 悬停仅切换色块 |
| 图片 | `image-rendering: pixelated` |
| 过渡 | ❌ 禁用 CSS transition/animation 平滑, 用 step() 或帧切换 |
